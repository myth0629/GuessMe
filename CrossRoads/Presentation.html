1. 프로젝트 개요

장르: 텍스트 기반 딜레마 시뮬레이션 / 인터랙티브 스토리텔링

핵심 컨셉: 정해진 스크립트 없이, 플레이어의 선택에 따라 AI가 실시간으로 결과를 생성하고 시나리오를 전개하는 "무한한 자유도의 딜레마 게임"

2. 생성형 AI 활용 전략

본 프로젝트는 Google Gemini 2.5 Flash Lite 모델을 활용하여, 정적인 게임 룰과 동적인 AI 스토리텔링을 결합했습니다.

2.1. AI의 역할: "시스템이자 작가"

상황 묘사

현재 상황과 위기를 3문장의 뉴스 속보 톤으로 건조하고 극적으로 묘사합니다.

선택지 생성

플레이어를 괴롭히는 4가지 딜레마 선택지를 실시간으로 생성합니다.

결과 판정

플레이어의 선택이 게임 내 수치(안정성, Stability)에 미치는 영향을 계산합니다.

2.2. 기술적 구현 방식

  • 모델 선정: gemini-2.5-flash-lite
    • 선정 이유: 게임 플레이의 흐름을 끊지 않는 빠른 응답 속도와 반복적인 턴 진행에 적합한 비용 효율성.
  • 프롬프트 엔지니어링:
    • Context Injection: 대화 히스토리 대신 현재의 GameState(턴, 안정도, 이전 선택 등)를 JSON으로 매 턴마다 주입하여 AI가 상황을 완벽히 인지.
    • Persona: 테마별(AI 통치자, 아파트 대표, 변호사)로 다른 "Role"을 부여하여 일관된 톤앤매너 유지.
    • Structured Output: Unity 처리를 위해 반드시 JSON 포맷으로만 응답하도록 강제.

2.3. 주요 기능

멀티 테마 시나리오

하나의 코드베이스로 프롬프트 내의 Theme Guideline만 교체하여 다양한 게임 경험 제공.

동적 난이도 조절

AI에게 "안정성 변화폭을 ±20으로 제한하라"는 수치 가이드를 제공하여 밸런싱 유지.

엔딩 생성

게임 종료 시, 플레이어의 여정을 요약하고 평가하는 서사적인 "엔딩 리포트" 작성.

2.4. 멀티미디어 확장

생성형 AI 이미지 사용

상황 묘사 텍스트를 기반으로, 현재 씬의 분위기에 맞는 배경 및 인물 이미지를 실시간으로 생성하여 시각적 몰입감을 극대화합니다.

Suno AI 배경음악 생성

Suno AI를 활용하여 각 테마(SF, 재난, 법정)와 현재 상황(긴장, 평화, 위기)에 최적화된 배경음악(BGM)을 생성 및 적용합니다.

3. 개발 중 직면한 문제와 해결

문제 1: AI 응답의 불확실성

현상: AI가 가끔 JSON 형식을 깨거나, 게임 룰에 없는 수치를 반환.

해결: 프롬프트에 "Step-by-Step Logic"을 명시하여 사고 과정을 강제하고, 클라이언트(C#)에서 JSON 파싱 실패 시 재시도하거나 기본값을 처리하는 Fallback 로직 구현.

문제 2: 문맥 유지

현상: 턴이 지날수록 앞선 설정을 잊어버림.

해결: 매 턴마다 GameState 전체를 다시 입력으로 넣어주어, AI가 "기억"에 의존하지 않고 "현재 데이터"를 보고 판단하게 함.

4. 결론 및 향후 발전 방향

이 프로젝트는 생성형 AI가 게임 개발의 리소스(스크립트 작성 등)를 획기적으로 줄여줄 뿐만 아니라, 그 자체로 새로운 게임 메커니즘이 될 수 있음을 증명했습니다.

향후에는 텍스트뿐만 아니라 상황에 맞는 이미지 생성이나 NPC와의 자유 대화 기능으로 확장 가능합니다.
